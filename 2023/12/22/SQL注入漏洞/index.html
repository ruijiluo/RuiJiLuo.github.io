一．SQL Inject漏洞概述

1．原理概述

  • 数据库注入漏洞，主要是开发人员在构建代码时，没有对输入边界进行安全考虑，导致攻击着可以通过合法的输入点提交一些精心构造的语句，从而欺骗后台数据库对其进行执行，导致数据库信息泄漏的一种漏洞。

  • SQL Injecti漏洞攻击流程:

    1. 注入点探测
      • 自动方式：使用web漏洞扫描工具，自动进行注入点发现
      • 手动方式：手工构造sql injecti测试语句进行注入点发现
    2. 信息获取( 通过注入点取期望得到的数据 )
      • 环境信息：数据库类型，数据库版本，操作系统版本，用户信息等。
      • 数据库信息：数据库名称数据库表，表字段，字段内容（加密内容破解
    3. 获取权限
      • 获取操作系统权限：通过数据库执行shell,上传木马等
2．常见注入点类型

  • 常见注入点类型

    • 数字型 后端代码形如：user_id=$id

    • 字符型 后端代码形如：user_id=’$id’ or ＂$id＂

    • 搜索型　　后端代码形如：test LIKE ‘%{$_GET[‘search’]}%’＂

2.基于函数报错的的方式来进行利用

  • 技巧思路：在MYSQL中使用一些指定的函数来制造报错，从而从报错信息中获取设定的信息。

    ​ select/insert/update/delete都可以使用报错来获取信息。

  • 背景条件：后台没有屏蔽数据库报错信息，在语法发生错误时会输出在前端。

  • 常用的报错函数:

    • updatexml()

      ​ 函数作用：改变（查找并替换）XML文档中符合条件的节点的值。
      ​ 语法：UPDATEXML(xml_document,XPathstring,new_value)
      ​ 第一个参数：fiedname是String格式，为表中的字段名。
      ​ 第二个参数：XPathstring(Xpath格式的字符串)，
      ​ 第三个参数：new_value,String格式，替换查找到的符合条件的

      ​ 注: 当XPathstring为函数的返回值时,会先执行函数,如果返回值不合规就将返回值通过报错方式显示到前端!

    • extractvalue()

      ​ 函数作用：从一个使用xpath语法的xml字符串中提取一个值。

      ​ 语法：extractvalue(xml_frag, xpath_expr)

      ​ xml_frag：xml文档对象的名称，是一个string类型。
      ​ xpath_expr：使用xpath语法格式的路径。

      ​ SQL报错注入的应用：当使用extractvalue(xml_frag, xpath_expr)函数时，若xpath_expr参数不符合xpath格式，就会报错。

      ​ 而符号(ascii编码值：0x7e)是不存在xpath格式中的， 所以一旦在xpath_expr参数中使用符号，就会产生xpath syntax error (xpath语法 错误)，通过使用这个方法就可以达到报错注入的目的。

    • floor()

      ​ 函数作用：向下取整
      ​ rand() 函数，取随机数，若有参数x，则每个x对应一个固定的值，如果连续多次执行会变化，但是可以预测
      ​ floor( rand( 0 ) * 2 ) 产生的随机序列为011011…
      ​ SQL报错注入的应用: 利用数据库表主键不能重复的原理，使用 GROUP BY 分组，产生主键key冗余，导致报错

四. SQL Injecti漏洞-盲注

1.boolian base原理及测试
  • 原理：
    • 攻击者通过构造SQL语句，向数据库发送一系列的布尔查询（True或False）。
    • 通过观察应用的响应，攻击者可以推断查询的结果是否为真或假，从而逐步猜测数据库中的数据。

  • 输入单引号,发现不会报错

  • 输入已知name:kobe,查询数据

  • 构建payload,判断是否存在sql注入漏洞

  • 以得到数据库名为目的构建payload,先获得数据库名长度为7

  • 构建payload,先获得数据库name的第一个字母为p，后续可通过手工或sqlmap或脚本得到完全数据库name

2.time base原理及测试

  • 原理：

    • 攻击者通过构造SQL语句，向数据库发送一系列的延时查询，通过观察应用的响应时间来判断查询是否成功。

    • 常见的是使用SLEEP()函数来引起延时。

  • time base只有同一种回显，可以通过浏览器网络部分中的时间线判断构建的payload的sql部分是否执行

  • 构建payload,先获得数据库name的第一个字母为p，后续可通过手工或sqlmap或脚本得到完全数据库name

3.DNS外带注入(OOB)原理

DNS外带注入(Out-of-Band, OOB)是一种通过其他传输通道窃取数据的技术，它利用数据库服务器发起DNS查询请求，将数据通过域名解析过程外带出来。

核心原理

    1. UNC路径触发DNS查询：Windows系统中的UNC路径(如\example.comshare)会自动触发DNS解析
    1. 数据拼接域名：将查询结果拼接到子域名中
    1. DNS日志捕获：通过监控DNS服务器日志获取外带数据

六.宽字节注入

  • 原理
    php通过转义’为\‘来避免’提前结束sql查询，但数据库使用的gbk编码会把两个字节转为一个汉字，可以使用一个高位字节连接\，使得\被转为汉字留下’，这样就使得’绕过了php转义
  • 利用方式
    使用%df’可以闭合引号（%df’–php–>%df\‘–数据库gbk–>[汉字]’）
  • 修复方案
    使用 mysql_set_charset(“utf8mb4”)指定字符集

九. sqli常见防范措施

一些常见的防范措施：

  1. 对输入进行严格转义和过滤。
  2. 使用预处理和参数化语句。
  3. 限制应用程序用户的权限。
  4. 更新和修补应用程序和数据库，以保护它们免受网络漏洞的影响。
  5. 部署Web应用程序防火墙（WAF）以识别和仅接受给定字段的允许输入。
  6. 对用户输入进行身份验证，以过滤恶意制作的外部输入。
  7. 实时监控关键数据库和Web服务器。
