暴力破解

发表于2023-12-15|更新于2023-12-15
|阅读量:

一.Burte Force（暴力破解）概述

  • “暴力破解”是一攻击具手段，在web攻击中，一般会使用这种手段对应用系统的认证信息进行获取。 其过程就是使用大量的认证信息在认证接口进行尝试登录，直到得到正确的结果。 为了提高效率，暴力破解一般会使用带有字典的工具来进行自动化操作。

  • 理论上来说，大多数系统都是可以被暴力破解的，只要攻击者有足够强大的计算能力和时间，所以断定一个系统是否存在暴力破解漏洞，其条件也不是绝对的。 我们说一个web应用系统存在暴力破解漏洞，一般是指该web应用系统没有采用或者采用了比较弱的认证安全策略，导致其被暴力破解的“可能性”变的比较高。 这里的认证安全策略, 包括：

  1. 是否要求用户设置复杂的密码；

  2. 是否每次认证都使用安全的验证码（想想你买火车票时输的验证码～）或者手机otp；

  3. 是否对尝试登录的行为进行判断和限制（如：连续5次错误登录，进行账号锁定或IP地址锁定等）；

  4. 是否采用了双因素认证；

二.BF漏洞分析

  • BF漏洞的产生

    • 弱密码策略： 如果系统允许用户设置弱密码，或者管理员默认使用弱密码，那么攻击者可以通过暴力破解方法很容易地尝试所有可能的密码组合，从而获得访问权限。

    • 未锁定账户： 如果系统在多次失败的登录尝试后不锁定账户或采取其他保护措施，攻击者可以继续进行暴力破解攻击，尝试不同的用户名和密码组合。

    • 缺乏验证码或二次验证： 如果系统没有实施验证码或二次验证等机制，攻击者可以通过自动化工具大规模尝试不同的用户名和密码组合，而不受到验证码或其他防御机制的干扰。

    • 漏洞或错误的身份验证实现： 一些系统可能存在身份验证实现上的漏洞，使攻击者能够绕过正常的认证过程。这可能包括对密码哈希的弱加密、错误的密码存储实践等。

  • 判断表单是否可以BF

    • 密码策略： 查看网站的密码策略，了解密码的复杂性和长度要求, 弱密码策略可以通过特定方式生成字典。
    • 账户锁定机制： 检查网站是否实施了账户锁定机制，即在一定次数的登录失败后是否会锁定用户账户。
    • 登录失败提示： 检查登录失败时网站返回的错误信息。如果错误信息明确指示了用户名或密码错误，可以利用这些信息来进行单密码或单用户名BF。
    • 验证码或Token： 检查网站在登录尝试次数达到一定阈值时是否会要求用户输入验证码,检查网站在登录页表单是否有提交Token,(验证码是否一次使用,是否过期销毁,验证码是否是以明文方式储存在前端)。

        • 注意使用以上方式时要将Payload Encoding中的URL-encode these characters(URL转译)关闭或者将URL转译中 & 和 = 去掉(防止 Payload 中出现 % )

    3. 验证码或Token或Cookie前端明文保存（展示Token的例子）

      • 通过burp suite抓包可以发现，数据包中提交数据包含了Token，在网页中查看源码可以发现登录下隐藏了明文存储的Token

      • 使用Repeater重放请求，发现每一次请求后相应数据包都会附带新的Token

      • 可以使用Grep-Extract提取相应中的Token用于下一次破解

        1. 攻击方式为Pitchfork，Payload(1)为 §u&password=p§ ,Payload(2)为 §50590657c3bebcda94044156160§

        2. Payload(1)模式为Custom iterator, 有效载荷选项(Payload Options)与前文一至

        3. Payload(2)模式为Recursive grep, 有效载荷选项(Payload Options)通过Grep-Extract提取(Initial payload for first request为第一次爆破的Token,可以不设置)

        4. Resource Pool中设置线程数(Maximum concurrent requests)为 1 (因为要用到相应数据), Options中Redirections将重定向设置为 总是(Always), 然后Start attack
