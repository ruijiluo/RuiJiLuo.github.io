FRP 学习笔记

1. 工具概述

FRP（Fast Reverse Proxy）是一款开源的高性能内网穿透工具，由 fatedier 用 Go 语言开发。它能够将内网服务通过公网服务器暴露到外部网络，让外网用户能够访问内网的 Web 服务、SSH、远程桌面等资源。

核心理念：利用一台有公网 IP 的服务器作为“中转站”，让内网机器主动建立连接，外部请求通过这个“中转站”转发到内网，从而实现穿透 NAT（网络地址转换）设备的效果。

2. 核心概念与架构

2.1 核心组件

FRP采用经典的 C/S（客户端/服务器）架构，由两个核心组件构成：

组件 简称 部署位置 作用
FRPS 服务端 有公网IP的云服务器 监听外部请求，转发给客户端
FRPC 客户端 内网需要暴露的机器 主动连接服务端，注册代理服务

2.2 工作原理

工作流程：

  1. 注册阶段：内网FRPC启动后，主动向公网FRPS发起连接并注册可用服务
  2. 监听阶段：FRPS在公网监听配置的端口，等待外部请求
  3. 转发阶段：用户访问FRPS的公网端口，FRPS将请求通过已建立的隧道转发给FRPC
  4. 响应阶段：FRPC将请求交给内网服务处理，并将结果原路返回

核心优势：整个过程不需要在内网路由器上配置端口转发，也不需要公网IP。

2.3 代理类型概览

FRP支持多种代理类型，以适应不同的穿透场景：

类型 功能 适用场景 特殊要求
TCP 纯粹的TCP端口映射 SSH、RDP、数据库、游戏 无
UDP UDP端口映射 DNS、VPN、语音通信 无
HTTP HTTP应用转发 Web网站、API服务 需要域名或公网IP
HTTPS HTTPS流量转发 需要SSL的Web服务 需要域名和证书
STCP 安全TCP代理 点对点访问，不在服务端暴露端口 访问端也需要部署FRPC
SUDP 安全UDP代理 点对点UDP通信 访问端也需要部署FRPC
XTCP P2P打洞穿透 大流量传输，流量不经过服务端 两端都部署FRPC，NAT类型友好

9. 故障排查

9.1 常见问题

问题现象 可能原因 解决方案
login to server failed: EOF TLS配置不匹配或网络不通 检查服务端端口是否开放，尝试禁用TLS测试
authorization failed Token不一致 检查服务端和客户端token是否相同
连接超时 防火墙未放行端口 检查云安全组和服务器防火墙
可以连接但无法传输数据 网络不稳定或代理类型错误 检查代理类型配置，尝试启用加密传输
XTCP打洞失败 NAT类型不支持P2P 改为STCP模式
remotePort already used 端口冲突 更换remotePort或停止占用端口的服务

  • 旧版本（v0.52前）：使用 frps.ini / frpc.ini，配置格式 [common]
