java反序列化漏洞

1.java反序列化原理:

  • Java 序列化机制提供了一种将对象的状态转换为字节流的方式，同时也可以将字节流重新转换为对象。这一过程涉及两个主要操作：序列化（serialization）和反序列化（deserialization）

  1. 序列化：

    • 使用 ObjectOutputStream 的 writeObject 方法将对象写入输出流。
    • 序列化过程会写出对象的类信息、类的序列化版本 UID（serialVersionUID）和对象的非瞬态字段（non-transient fields）。

  2. 反序列化：

    • 使用 ObjectInputStream 的 readObject 方法从输入流读取对象。

    • 反序列化过程会读取类信息、类的序列化版本 UID，并重建对象的非瞬态字段。

2.漏洞原理:

  • 简单来说就是java反序列化的过程中会通过java反射机制调用序列化对象对应的类中的readObject()方法进行反序列化,如果序列化的对象的类实现了serialization接口,同时该类中的**readObject()**执行了恶意的操作,并且后端对序列化数据没有检验和过滤,攻击者就可以进行反序列化攻击。

3.readObject()的调用机制

  • 当java调用 ObjectInputStream 的 readObject() 时，Java 会执行以下步骤：

    1. 读取类信息：ObjectInputStream 首先从输入流中读取对象的类信息，包括类名和序列化版本 UID。
    2. 检查类是否实现了 Serializable 接口：如果类没有实现 Serializable 接口，会抛出 NotSerializableException。
    3. 实例化对象：Java 会创建对象的一个新实例。此时，不会调用对象的构造函数。
    4. 恢复对象的非瞬态字段：Java 会从输入流中读取对象的字段，并将它们的值设置到新实例上。
    5. 调用 readObject() （如果存在）：如果被反序列化的类定义了一个私有的 readObject 方法，Java 会通过反射机制调用这个方法来进行自定义的反序列化逻辑。

4.漏洞利用链

  • 利用链的概念 ：在现实生成环境中我们不能保证别人的项目中存在User这个类，所有我们要从javajdk或第三方库中找到实现了Serializable，同时又在readObject中进行了危险操作的类进行利用，而满足这些条件的类成为利用链，注意这里提到的是链，意味着这些存在问题的类并不是单一的一个，而是由多个类组合造成的。

5.Ysoserial

  • 在Ysoserial中已经提供好了很多这种符合利用条件的类(利用链)，并且给出了相应存在问题的依赖版本，只要项目里引入了这些对应版本的依赖，就会存在这些有问题的类。
