Shiro-721 Padding Oracle攻击完整笔记

1. 漏洞本质

  • 漏洞编号：CVE-2019-12422
  • 影响版本：Shiro < 1.4.2
  • 漏洞类型：Padding Oracle Attack
  • 前提条件：攻击者需要有一个有效的RememberMe Cookie

八、总结要点

核心思想

  1. 旁路攻击：利用错误信息，而非算法弱点
  2. 数学基础：CBC的线性特性 P = Intermediate ⊕ C_prev
  3. 递进破解：从后往前，依赖前一步结果
  4. 最终控制：控制IV等于控制解密结果

攻击条件

  • ✅ 使用CBC模式
  • ✅ 服务端返回不同的padding错误
  • ✅ 能够重复发送请求
  • ✅ 有一个合法密文（Shiro-721需要）

修复关键

  • 升级到Shiro 1.4.2+
  • 使用随机密钥
  • 改用AEAD模式
  • 统一错误响应

一句话总结：Shiro-721是通过Padding Oracle攻击，在不知道密钥的情况下，利用已有合法Cookie伪造恶意Cookie，最终实现反序列化RCE的漏洞。
