Mimikatz 完全学习笔记：从入门到精通

1. 工具概述

1.1 什么是Mimikatz？

Mimikatz是一款开源的Windows安全审计工具，它能够从Windows操作系统内存中提取明文密码、哈希、PIN码和Kerberos票据。

核心理念：Windows为了支持单点登录（SSO），会在LSASS（Local Security Authority Subsystem Service）进程中缓存用户的认证凭证。Mimikatz就是利用这一点，读取这些内存数据。

1.2 为什么这么重要？

场景 价值
渗透测试 快速获取域控权限，横向移动
红队演练 模拟高级攻击手法
蓝队培训 理解攻击原理，完善检测规则
应急响应 分析被入侵系统，追溯攻击链（DC取证）

Mimikatz被称为内网渗透的“瑞士军刀”，是每个Windows安全研究者的必修课。

4.5 lsadump::lsa /patch

功能：直接从LSASS进程内存中读取LSA策略数据，获取凭证。

风险：直接修补当前运行的LSASS进程的代码段，可能导致LSASS崩溃触发主机重启（1分钟倒计时）。

4.6 lsadump::dcsync（DCSync攻击）

功能：模仿域控制器行为，从目标域控请求复制账户密码数据。这是Mimikatz最强大的远程凭证获取技术！

原理：利用目录复制服务（DRS）远程协议，向域控发送DSGetNCChanges复制请求。这是微软官方支持的Active Directory复制机制！

权限要求：Administrators组、Domain Admins组、Enterprise Admins组、拥有目录复制权限的自定义委派。

优势：不需要在域控上执行代码，可以远程操作！

5. 票据操作

5.1 查看和导出票据

5.2 黄金票据（Golden Ticket）

功能：利用krbtgt账户的NTLM哈希，伪造任意用户的TGT票据。黄金票据是Mimikatz最强大的攻击之一，被称为“域控通行证”。

参数说明：

参数 说明 示例
/user 要伪造的用户名 Administrator
/domain 域名FQDN contoso.com
/sid 域的SID（不含RID） S-1-5-21-...
/krbtgt krbtgt的NTLM哈希 36f1e3f2...
/ptt 生成后立即注入

黄金票据为什么强大？

  • 持久性：默认有效期为10年，krbtgt密码不变则永远有效
  • 任意用户：可以伪造任何用户，甚至不存在的用户
  • 无需验证：KDC不验证TGT中的用户是否存在

5.3 白银票据（Silver Ticket）

功能：伪造特定服务的TGS票据，访问特定服务。与黄金票据的区别：黄金票据伪造TGT（可访问任何服务），白银票据伪造TGS（只能访问特定服务）。

常用服务类型：

服务代码 对应的服务 能做什么
cifs 文件共享服务 访问C$、Admin$、IPC$
host 计划任务服务 schtasks创建/执行计划任务
http Web服务 访问需要Kerberos认证的Web应用
ldap LDAP服务 查询AD目录
wsman WinRM服务 远程PowerShell
rpcss RPC服务 远程过程调用
mssql MSSQL服务 SQL Server管理

优点：直接伪造ST，不会在KDC产生事件4769（请求Kerberos服务票据），更隐蔽。

6. 哈希传递（Pass-the-Hash）

原理：Windows认证机制中，NTLM哈希是认证的核心凭证。只要能提供正确的哈希，就能通过NTLM挑战-响应认证。

限制：Windows Server 2012 R2之后默认不存储明文密码，但NTLM哈希仍可在内存中找到。

8. 横向移动与权限维持

8.1 票据传递（Pass-the-Ticket）

场景：如果抓到了域管理员曾经登录过的票据，注入后即可获得域管理员权限访问域控。

8.2 Skeleton Key（万能钥匙）

功能：在域控LSASS中注入万能钥匙，允许任何用户使用特定主密码登录。

特点：重启后失效（需重新注入），不影响原密码，极难检测。

8.3 启动子进程（继承票据）

原理：子进程默认继承父进程的访问令牌（Token），包括其中的Kerberos票据缓存。

9.3 为什么Mimikatz容易被杀？

  • 特征字符串："sekurlsa::logonpasswords"、"privilege::debug"
  • API调用序列：NtQuerySystemInformation → OpenProcess → ReadProcessMemory
  • LSASS内存读取行为：正常程序不会读取LSASS进程内存

9.5 防御措施

措施 效果
启用Credential Guard 隔离LSASS，使用VBS保护
设置RunAsPPL=1 阻止非受保护进程读取LSASS
禁用WDigest 明文密码不存储在内存中
定期修改krbtgt密码 使黄金票据失效（需修改两次！）
最小权限原则 减少高权限凭证暴露

10. 常见错误与排查

问题 可能原因 解决方案
privilege::debug失败 权限不足 以管理员身份运行
0xc0000061 令牌被UAC过滤 创建新的高完整性进程
sekurlsa::logonpasswords无输出 没有用户登录 RDP登录后再试
被杀毒软件拦截 特征被检测 使用内存加载或混淆
票据注入失败 权限不够 需要高权限运行
DCSync失败 权限不够 需要域管理员权限
sekurlsa::minidump解析失败 dmp版本不兼容 换用旧版本Mimikatz或pypykatz
lsadump::lsa /patch导致重启 LSASS崩溃 用离线分析替代
